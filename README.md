# AI Product Recommendation System

## Overview

AI Product Recommendation System is a web application that allows users to describe their product preferences in natural language and receive personalized product recommendations generated by a Large Language Model (LLM).

The application combines a Django backend, a lightweight HTML/CSS/JavaScript frontend, and the OpenRouter API to provide intelligent product recommendations from a predefined product catalog.

Example user queries:

* "I want a phone under $500"
* "Recommend a smartphone with a great camera"
* "Suggest an Android phone for everyday use"

The AI analyzes the user's requirements, compares them against the available product catalog, and returns the most suitable recommendations with explanations.

## How It Works

### 1. Product Catalog

The application maintains a predefined list of products containing:

* Product ID
* Product Name
* Price
* Description

### 2. User Input

The user enters a preference in natural language.

Example:

```text
I want a phone under $500
```

### 3. Prompt Generation

The backend combines:

* Product catalog
* User preference
* Recommendation instructions

into a structured prompt.

### 4. AI Processing

The prompt is sent to the OpenRouter API.

The language model evaluates available products and generates recommendations based only on products present in the catalog.

### 5. Response Delivery

The recommendation is returned as JSON and displayed dynamically in the frontend.

---

## API Endpoint

### Generate Recommendations

**Endpoint**

```http
POST /recommend/
```

**Request**

```json
{
  "preference": "I want a phone under $500"
}
```

**Response**

```json
{
  "recommendations": "Samsung A55 is a strong match because it is within the budget and offers excellent value for everyday use."
}
```

## Local Setup

### Clone Repository

```bash
git clone <repository-url>
cd ai_product_recom
```

### Install Dependencies

Using uv:

```bash
uv sync
```

Or using pip:

```bash
pip install -r requirements.txt
```

### Run Migrations

```bash
python manage.py migrate
```

### Start Development Server

```bash
python manage.py runserver
```

Application will be available at:

```text
http://127.0.0.1:8000
```

## Deployment

### Backend

### Frontend

The static frontend is deployed on Vercel.

The frontend communicates with the deployed Django API endpoint to retrieve recommendations.
